# 开发规范

## 分支与提交

- 功能开发使用独立分支。
- 提交信息保持简洁明确。
- 一个提交只做一类改动。

## 后端规范

- 所有 API 返回统一响应结构。
- 所有写接口必须校验权限和参数。
- 业务复杂计算放到服务层或异步任务。
- 数据模型字段命名要可追溯和可迁移。
- 提交前至少执行 `make test` 和 `python3 -m py_compile $(rg --files backend -g '*.py')`。

## 前端规范

- 页面按“工作台、列表、详情、编辑”组织。
- 统一使用 `src/api` 封装网络请求。
- 登录态和 token 只存储在本地安全位置。
- 页面空状态、错误状态、加载状态必须补齐。

## 数据规范

- 食材名称使用标准名。
- 记录保留来源和时间戳。
- 外部 API 数据需要归一化后再入库。
