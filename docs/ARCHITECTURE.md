# 系统架构

## 分层

- 表现层：Vue 3 + Element Plus + ECharts
- 接口层：Django REST Framework
- 业务层：推荐、营养分析、饮食记录、社区治理
- 数据层：PostgreSQL、Redis、对象存储
- 任务层：Celery

## 数据流

1. 用户在前端登录。
2. 前端携带 JWT 调用后端 API。
3. 后端执行业务校验、读写 PostgreSQL。
4. 高成本任务交给 Celery。
5. 外部 API 用于补全营养库和菜谱信息。

## 关键设计

- 菜谱、营养、行为、记录分表存储。
- 推荐采用规则过滤 + 内容召回 + 行为排序。
- 饮食记录和目标追踪强调可解释性。
- 社区内容与核心业务解耦，支持治理。
